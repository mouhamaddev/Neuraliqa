Greedy Algorithm

A greedy algorithm is a method of solving a problem that chooses the best solution available. It builds up a solution piece by piece, always choosing the next piece that offers the most obvious and immediate benefit.

Greedy algorithm is not concerned about whether the current best outcome will lead to the overall best result.

Let’s give a real time example. John has some 5¢, 10¢, 20¢ and 25¢ coins.

He is going to give Mike an amount of 30¢. So what a greedy algorithm is going to pick the minimal number of coins. It will look at the max value of a coin, which is 25¢ here, and pick one piece of it.

Now we have 25¢ and we still need 5¢.

Then the greedy algorithm will look at the second max coin which is 20¢, and since 25¢ + 20¢ = 45¢ bigger than 30¢, so it will look at the second one which is 10¢. Same idea here. Then it will look at the 5¢ and pick it since 25¢ + 5¢ = 30¢.

In this example, the greedy algorithm picked an amount of 30¢ with just two coins : 25¢ + 5¢, which is the minimum number of coin available.

But, a greedy algorithm is not always the best idea to chose. Let’s give another example.

In this example, John has the same coins, but he is going to give Mike 40¢.

So a greedy algorithm will look at the max amount, which is 25¢, then 10¢, then 5¢.

And that’s 40¢ with 3 coins.

Here is when a greedy algorithm fails to choose the smallest amount of coins, since we could have chosen two 20¢ coins and this is already a 40¢.

Let’s implement the coin example in a python code :

def coinfunc(total, coins):
    T = total
    coins.sort()
    index = len(coins)-1
    while True:
        coinValue = coins[index]
        if T >= coinValue:
            print(coinValue)
            T = T - coinValue
        if T < coinValue:
            index -= 1
        
        if T == 0:
            break

coins = [5,10,20,25]
coinfunc(40, coins)
