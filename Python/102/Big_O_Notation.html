Big O Notation

Big O notation is simplified analysis of an algorithms efficiency.

Big O notation is one of the most fundamental tools for computer scientists to analyze the cost of an algorithm. It describes the complexity of your code using algebraic terms. We only care here about time and memory.

Let's explain Big O in very simple words.

Let me ask you a question. We want to send mails from city to other. Which one is faster, internet sending information? or a mail truck?

Now you'll probably say, this is ridiculous! Are you serious? Of course internet is faster.

And that may be true. But it depends on the mails size. 200MB? yes internet is of course faster. But what if it was 10GB? The truck wins. That's because the truck has a constant amount of time. It takes 20 mins from a city to other whatever how many mails it has.

So the first rule of big o is : we only care about the forever time. Let's give an example.

Let’s say we have two functions. f(x) = x and f(x) = x^2. That’s the graph :

We can see that O(n) (which is f(x) = x) is dominating in the beginning. And also there’s an intersection point.

Above the intersection point we can clearly see that O(n^2) is dominating over O(n). So O(n^2) is more efficient, since it’s dominating in the forever time.

Another example is O(2n) (or any constant instead on 2).

O(n^2) will still dominate O(2n) at some point. And this is the second rule of Big O : we don’t look at constants and we consider O(2n) (or any other constant) as O(n).

Here’s some pseudocode :

Let n equal 0.
For each item in list :
	set n = n + 1.

Let’s break it down to simple lines and calculate Big O of this code.

Let n equal 0.

The first line is called primitive operation. And it runs at one unit of time (one unit could be 10 ms, one second, or one minute etc..). So the first line is O(1)

For each item in list :
	set n = n + 1.

The second line and the third is a loop. And it’s considered as O(n)

And as we said before, O(n) dominates O(1) so this simple code is considered as O(n).

Certain terms dominate others :

O(1) < (O log n) < O(n) < (O n log n) < O(n^2) < O(2^n) < O(n!)

More Examples :

Let’s give some more practical examples :

for x in range(a):
	for y in range(b):
		print(x + y) #O(1)

This code is two loops inside each other, and it’s a O(n^2).

x = 24 + 74 #O(1)

	print(x)               # => O(n)
